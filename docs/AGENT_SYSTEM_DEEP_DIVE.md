## 1. 什么是 Agent 系统

### 1.1 简单比喻

想象你有一个**超级助理**：
- 他能听懂你说的话（自然语言理解）
- 他能思考如何完成任务（推理规划）
- 他有很多工具可以使用（搜索、写代码、发消息...）
- 他能记住之前的对话（会话记忆）
- 他还能叫其他助理来帮忙（子代理）

这就是 **AI Agent**（智能代理）。

### 1.2 与普通聊天机器人的区别

```
┌─────────────────────────────────────────────────────────────────┐
│                     普通聊天机器人                                │
│                                                                  │
│   用户: "今天天气怎么样？"                                        │
│                    ↓                                             │
│              [LLM 模型]                                          │
│                    ↓                                             │
│   回复: "我无法获取实时天气信息..."                                │
│                                                                  │
│   ❌ 只能回答，不能行动                                           │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                      AI Agent                                    │
│                                                                  │
│   用户: "今天天气怎么样？"                                        │
│                    ↓                                             │
│              [LLM 模型] ──→ "我需要查询天气"                      │
│                    ↓                                             │
│              [调用天气工具]                                       │
│                    ↓                                             │
│              [获取结果: 晴天 25°C]                                │
│                    ↓                                             │
│              [LLM 模型] ──→ 组织回复                              │
│                    ↓                                             │
│   回复: "今天是晴天，气温 25°C，很适合出门！"                      │
│                                                                  │
│   ✅ 能思考 + 能行动 + 能回答                                     │
└─────────────────────────────────────────────────────────────────┘
```

```

                    │   AI Agent      │

                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
   ┌─────────┐        ┌─────────────┐      ┌──────────┐

```

## 2. 核心概念入门

### 2.1 关键术语表

| 术语 | 英文 | 含义 | 比喻 |
|------|------|------|------|
| Agent | Agent | 智能代理 | 你的 AI 助理 |
| Tool | Tool | 工具 | 助理的技能（搜索、写代码...）|
| Session | Session | 会话 | 一次完整的对话 |
| Turn | Turn | 轮次 | 一问一答 |
| Prompt | Prompt | 提示词 | 给助理的指令 |
| Context | Context | 上下文 | 对话的背景信息 |
| Compaction | Compaction | 压缩 | 总结过长的历史 |
| Skill | Skill | 技能 | 可插拔的能力模块 |
| Subagent | Subagent | 子代理 | 被召唤来帮忙的其他助理 |

### 2.2 消息角色

Agent 系统中有三种消息角色：

```
┌────────────────────────────────────────────────────────────────┐
│                       消息角色 (Roles)                          │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│   👤 User (用户)                                               │
│   └── 你发送的消息                                              │
│       例: "帮我搜索一下今天的新闻"                               │
│                                                                 │
│   🤖 Assistant (助手)                                          │
│   └── AI 的回复和思考                                           │
│       例: "好的，我来帮你搜索..."                                │
│                                                                 │
│   🔧 Tool (工具)                                               │
│   └── 工具的执行结果                                            │
│       例: "[搜索结果: 1. xxx 2. xxx ...]"                       │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 4.3 创建 Agent 会话的代码流程

```typescript
// 1. 准备资源
const resourceLoader = new DefaultResourceLoader({
  cwd: resolvedWorkspace,        // 工作目录
  agentDir,                      // Agent 配置目录
  settingsManager,               // 设置管理器
  additionalExtensionPaths,      // 扩展路径
});
await resourceLoader.reload();

// 2. 创建会话
const { session } = await createAgentSession({
  cwd: resolvedWorkspace,
  agentDir,
  authStorage: params.authStorage,      // 认证信息
  modelRegistry: params.modelRegistry,  // 模型注册表
  model: params.model,                  // 使用的模型
  thinkingLevel: "medium",              // 思考级别
  tools: builtInTools,                  // 内置工具
  customTools: allCustomTools,          // 自定义工具
  sessionManager,                       // 会话管理器
  settingsManager,
  resourceLoader,
});

// 3. 应用系统提示词
applySystemPromptOverrideToSession(session, systemPromptOverride);

// 4. 订阅事件
const subscription = subscribeEmbeddedPiSession({
  session: session,
  runId: params.runId,
  onBlockReply: async (payload) => {
    // 收到回复块
    await sendToChannel(payload.text);
  },
  onToolResult: async (result) => {
    // 工具执行完成
    console.log("Tool result:", result);
  },
});

// 5. 发送提示
await session.prompt("你好，请帮我搜索今天的新闻");
```

## 5. Agent 循环机制

### 5.1 什么是 Agent 循环

Agent 循环是 AI Agent 的**核心执行机制**，它让 Agent 能够：
1. 接收用户输入
2. 思考需要做什么
3. 调用工具执行
4. 根据结果继续思考或返回答案

### 5.2 循环流程图

```
                         ┌─────────────────┐
                         │    开始 Start   │
                         └────────┬────────┘
                                  │
                                  ▼

            │        │   (发送消息给模型)     │       │
            │        └───────────┬───────────┘       │
            │                    │                   │
            │                    ▼                   │
            │        ┌───────────────────────┐       │
            │        │   解析 LLM 响应        │       │
            │        └───────────┬───────────┘       │
            │                    │                   │
            │           ┌───────┴───────┐            │
            │           │               │            │

            │  ┌─────────────┐  ┌─────────────┐     │
            │  │ 执行工具     │  │ 返回最终    │     │
            │  │ (可能多个)  │  │ 回复给用户  │     │
            │  └──────┬──────┘  └─────────────┘     │
            │         │                              │
            │         ▼                              │
            │  ┌─────────────┐                      │
            │  │ 将工具结果   │                      │
            │  │ 加入消息列表 │                      │
            │  └──────┬──────┘                      │
            │         │                              │
            └─────────┴──────────────────────────────┘
                      │
                      │  继续循环
                      │
                      ▼
               (回到调用 LLM)


⚠️ 安全机制: 最大迭代次数限制 (默认 25 次)
```

### 5.3 详细步骤分解

```
步骤 1: 接收用户输入
━━━━━━━━━━━━━━━━━━━━

用户: "帮我查一下北京今天的天气，然后告诉我穿什么衣服合适"


步骤 2: 构建消息列表
━━━━━━━━━━━━━━━━━━━━

messages = [
  { role: "system",    content: "你是一个有帮助的助手..." },
  { role: "user",      content: "帮我查一下北京今天的天气..." },
]


步骤 3: 第一次调用 LLM
━━━━━━━━━━━━━━━━━━━━━━

LLM 返回:
{
  content: "我来帮你查询天气",
  tool_calls: [{
    name: "web_search",
    arguments: { query: "北京今天天气" }
  }]
}


步骤 4: 执行工具
━━━━━━━━━━━━━━━━

工具执行结果:
"北京今天天气：晴，气温 15-25°C，西北风 3 级"


步骤 5: 将结果加入消息
━━━━━━━━━━━━━━━━━━━━━━

messages = [
  { role: "system",    content: "你是一个有帮助的助手..." },
  { role: "user",      content: "帮我查一下北京今天的天气..." },
  { role: "assistant", content: "我来帮你查询天气", tool_calls: [...] },
  { role: "tool",      content: "北京今天天气：晴..." },
]


步骤 6: 第二次调用 LLM
━━━━━━━━━━━━━━━━━━━━━━

LLM 返回:
{
  content: "北京今天天气晴朗，气温 15-25°C。建议穿着：
            - 外套或薄毛衣（早晚温差大）
            - 长裤
            - 可带一件薄外套以防风",
  tool_calls: []  // 没有工具调用
}


步骤 7: 返回最终回复
━━━━━━━━━━━━━━━━━━━━

用户收到: "北京今天天气晴朗..."
```

## 8. 流式传输机制

### 8.1 为什么需要流式传输

```
┌────────────────────────────────────────────────────────────────────┐
│                     流式 vs 非流式对比                              │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │

│                                                                     │

│         ▼                                                           │
│    ┌─────────────────────────────────────────┐                     │
│    │          等待 10-30 秒...                │  😴 用户很焦虑      │
│    │          (LLM 生成完整回复)              │                     │

│                                                                     │
│  ✅ 流式 (边生成边显示)                                            │
│                                                                     │

│         │                                                           │
│         ▼                                                           │
```

```
│                                                                     │
│  ┌────────────────────────────────────────────────────────────┐    │
│  │  🟢 可忽略错误 (Ignorable)                                  │    │
│  │     - 非关键功能失败                                        │    │
│  │     - 记录日志，继续运行                                    │    │
```

```rust
        &self,
        request: AgentRunRequest,
        delta_tx: mpsc::Sender<StreamDelta>,
    ) -> Result<AgentRunResult> {
        // 1. 构建系统提示词
        let system_prompt = build_system_prompt(...);
        
        // 2. 加载历史
        let history = self.session_store.get_history(&request.session_key).await?;
        
        // 3. 构建消息列表
        let mut messages = vec![
            LlmMessage::system(&system_prompt),
        ];
        messages.extend(history);
        messages.push(LlmMessage::user(&request.message));
        
        // 4. Agent 循环
        for _ in 0..MAX_ITERATIONS {
            // 调用 LLM
            let response = self.call_llm(&request.model, &messages, &delta_tx).await?;
            
            // 检查工具调用
            if response.tool_calls.is_empty() {
                // 保存历史并返回
                self.session_store.append_turn(...).await?;
                return Ok(AgentRunResult { ... });
            }
            
            // 执行工具
            for tool_call in &response.tool_calls {
                let result = self.tool_registry
                    .execute(&tool_call.name, &tool_call.input)
                    .await;
                
                // 发送事件
                delta_tx.send(StreamDelta::ToolResult { ... }).await?;
                
                // 添加到消息
                messages.push(LlmMessage::tool(&result));
            }
        }
        
        Err(Error::Agent("Max iterations reached".into()))
    }
}
```

### 13.3 工具 Trait 定义

```rust
// openclaw-agent/src/tools/mod.rs

/// Agent 工具 trait
#[async_trait]
pub trait AgentTool: Send + Sync {
    /// 工具名称
    fn name(&self) -> &str;
    
    /// 工具描述 (给 LLM 看)
    fn description(&self) -> &str;
    
    /// 参数 JSON Schema
    fn parameters_schema(&self) -> Value;
    
    /// 执行工具
    async fn execute(
        &self,
        params: Value,
        context: &ToolContext,
    ) -> Result<String>;
}

/// 工具注册表
pub struct ToolRegistry {
    tools: HashMap<String, Arc<dyn AgentTool>>,
}

impl ToolRegistry {
    pub fn new() -> Self {
        let mut registry = Self { tools: HashMap::new() };
        
        // 注册内置工具
        registry.register(Arc::new(WebSearchTool::new()));
        registry.register(Arc::new(WebFetchTool::new()));
        registry.register(Arc::new(ImageGenerationTool::new()));
        registry.register(Arc::new(MemorySearchTool::new()));
        registry.register(Arc::new(BrowserTool::new()));
        registry.register(Arc::new(CronTool::new()));
        
        registry
    }
}
```
